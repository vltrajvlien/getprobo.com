The Probo CLI (`prb`) is a command-line tool for interacting with the Probo platform. It lets you manage frameworks, controls, risks, findings, and more â€” directly from your terminal.

## Installation

### Homebrew (macOS and Linux)

```bash
brew install getprobo/tap/prb
```

### From source

```bash
go install github.com/getprobo/probo/cmd/prb@latest
```

### Verify installation

```bash
prb version
```

## Getting started

<Steps>

1. **Authenticate with your Probo instance**

   ```bash
   prb auth login
   ```

   This opens an interactive prompt where you provide your hostname, authentication token, and organization.

2. **Verify your authentication**

   ```bash
   prb auth status
   ```

3. **List your frameworks**

   ```bash
   prb framework list
   ```

</Steps>

## Global flags

| Flag | Environment Variables | Description |
|---|---|---|
| `--no-interactive` | `PROBO_NO_INTERACTIVE=1`, `CI=true`, `DEBIAN_FRONTEND=noninteractive`, `TERM=dumb` | Disable interactive prompts |
| `--no-color` | `NO_COLOR`, `TERM=dumb` | Disable ANSI color output |

## Common patterns

**Output formatting** â€” Most `list` and `view` commands support `--output json` or `--json` to get machine-readable output instead of the default styled table.

**Pagination** â€” `list` commands accept `--limit` (or `-L`, default 30), `--order-by`, and `--order-direction` (ASC/DESC) to control result sets.

**Organization scoping** â€” Commands that operate within an organization accept `--org` to specify which organization to target. If omitted, the default organization from `prb auth login` is used.

**Interactive mode** â€” Several commands support interactive prompts powered by terminal UI. These are automatically disabled in CI environments or when `--no-interactive` is set.
